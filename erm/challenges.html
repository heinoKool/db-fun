Praxis-Mission: ER-Modellierung

Du hast die Theorie im Skript verstanden? Jetzt geht's an die Praxis.

Hier sind einige Aufgaben. Deine Mission: Übersetze die Anforderungen in ein sauberes ER-Modell (UML-Notation).

Weitere Infoquellen: Wikipedia: ER‑Modell Video ER-Modell
Tools: draw.io / diagrams.net (Tipp: Online-Tool zum Zeichnen von ER‑Modellen)

Challenge 1: "Instakram"

Das Briefing: Du baust die Core-Features von Instagram nach. Im Zentrum stehen User, Posts, Kommentare, Likes und natürlich Follower.

Die Regeln (Anforderungen):

  • Ein User kann mehrere Posts erstellen. Ein Post gehört genau einem User.
  • Ein User kann viele Posts "liken". Ein Post kann von vielen Usern "geliked" werden.
  • Ein User kann viele Kommentare schreiben. Ein Post kann viele Kommentare haben.
  • Ein Kommentar gehört immer zu *genau einem* User UND *genau einem* Post.
  • (Die Knifflige): Ein User kann vielen anderen Usern folgen. Ein User kann von vielen anderen Usern gefolgt werden.

Challenge 2: "Kino"

Das Briefing: Du designst das System für eine große Kinokette, um deren Standorte, Säle und das laufende Programm zu verwalten. (Basiert auf PDF-Aufgabe B).

Die Regeln (Anforderungen):

  • Ein Ort (z.B. "Berlin") hat mehrere Kinos. Ein Kino ist aber in genau einem Ort.
  • Ein Kino (z.B. "CineStar Potsdamer Platz") hat mehrere Kinosäle.
  • Ein Film (z.B. "Dune 2") kann in vielen Kinos gezeigt werden.
  • Ein Kino kann viele Filme zeigen. (Vorsicht, Falle! Wie löst man das sauber auf?)
  • Eine Vorstellung (der eigentliche "Event") ist die Vorführung von *genau einem* Film in *genau einem* Saal zu einem bestimmten Datum/Uhrzeit.
  • Jeder Saal hat Attribute, die du mitspeichern sollst: `Name` (z.B. "Saal 8"), `Sitzplätze` (z.B. 250) und `Leinwandgröße_m2` (z.B. 50).

Challenge 3 (Hard): "Pokédex-Basis"

Das Briefing: Du bist Professor Eich! Du baust die Basis-Datenbank für den Pokédex. Es geht um die Pokémon-Spezies, ihre Typen und Attacken.

Die Regeln (Anforderungen):

  • Eine Spezies (z.B. "Bisasam", "Pikachu") hat eine Pokédex-Nummer und einen Namen.
  • Jede Spezies gehört zu (mindestens) einem Typ (z.B. "Pflanze", "Elektro").
  • Ein Typ kann zu vielen Spezies gehören. (z.B. "Flug" bei "Glurak" und "Taubsi").
  • Eine Spezies kann mehrere Attacken lernen.
  • Eine Attacke (z.B. "Tackle", "Donnerblitz") kann von vielen Spezies gelernt werden.
  • Jede Attacke hat aber *genau einen* Typ (z.B. "Tackle" ist Typ "Normal", "Donnerblitz" ist Typ "Elektro").
  • (Bonus): Eine Spezies kann sich zu einer anderen Spezies entwickeln (z.B. "Pikachu" → "Raichu").

Challenge 4 (Profi): "Pokédex-Liga"

Das Briefing: Du baust auf Challenge 3 auf. Jetzt kommen die Trainer, die individuellen Pokémon und die Arenen ins Spiel.

Die Regeln (Anforderungen):

  • (Basis): Du brauchst die Entität Trainer (z.B. "Ash", "Misty").
  • Ein Trainer kann viele Pokémon besitzen (seine "individuellen" Pokémon im Team).
  • Ein individuelles Pokémon gehört aber zu *genau einem* Trainer.
  • Jedes individuelle Pokémon ist von *genau einer* Spezies (aus Challenge 3).
  • (Arena): Du brauchst die Entität Arena (z.B. "Marmoria City Arena").
  • Jede Arena hat *genau einen* Trainer als Arenaleiter. Ein Trainer kann nur *eine* Arena leiten.
  • (Orden): Ein Trainer kann viele Orden von verschiedenen Arenen gewinnen.
  • Eine Arena kann ihren Orden an viele Trainer vergeben haben.
